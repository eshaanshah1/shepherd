# Shepherd ‚Äî agent-native macOS terminal

Shepherd is a native macOS terminal built on **libghostty** that treats **Claude
Code** sessions as first-class, tracked agents. It behaves exactly like a normal
terminal (iTerm/Ghostty-style) until you start `claude` in a tab ‚Äî then that tab
becomes an agent with a **live state** (working / blocked / need-to-check / idle /
error) surfaced in a sidebar, so you can run several at once and never babysit
them. When an agent needs you and you've switched apps, a notification + dock
badge pull you back.

**Status:** working v1 end-to-end (personal / OSS, single window). On `master`.
**Design doc:** [`SPEC.md`](SPEC.md). **Decisions:** [`.claude/adr/`](.claude/adr/) ‚Äî read these before changing anything load-bearing.

- `StopPolicy.swift` ‚Äî **pure model**: `applyEvent(...)` ‚Äî the whole hook‚Üístate lifecycle map + ordering guard + the background-agent `Stop`-suppression counter ([ADR 0014](.claude/adr/0014-background-agent-stop-suppression.md)), returning a `StateTransition`. `AgentStore.apply` is the AppKit shell around it (mirrors `SleepPolicy`/`SleepGuard`). - `SocketServer.swift` ‚Äî in-app unix-domain socket server (receives `{tab_id,event,detail}` ‚Äî `tab_id` is really the pane id).
- `SidebarView.swift` ‚Äî the tab list: a **custom `ScrollView` of rows (not `List`)**. - `ContentView.swift` ‚Äî sidebar + a resizable hairline divider + a ZStack of **every workspace's** tabs (each rendered via `SplitContainer`), opacity-gated so only the selected workspace's selected tab is visible/hit-testable while background-workspace surfaces stay mounted (agents keep running); switch **cross-fades** on `selectedWorkspaceID`. Feeds the content rect to the store so ‚åò‚å•-arrow focus moves can resolve geometric neighbors.
- `Workspace.swift` ‚Äî **pure model**: a `Workspace` (id, `userTitle`, `tabs`, `selectedTabID`; `displayName`/`aggregateState`/`reseedIfEmpty`) + pure free helpers (`locatePane`, `removingWorkspace`, `totalAttentionCount`). No AppKit ‚Äî in `ShepherdModelTests`.

- `SleepGuard.swift` ‚Äî `@MainActor` keep-awake controller: holds `pmset disablesleep`

## How to build & run

### 1. libghostty (only needed once, or to update)
Prereqs (see [ADR 0002](.claude/adr/0002-libghostty-build-on-macos-26.md)): brew `zig@0.15`, Xcode + Metal Toolchain.
```sh
./scripts/build-libghostty.sh    # clones Ghostty v1.3.1, builds, vendors the xcframework
```

### 2. The app
```sh
cd spike/seam1
xcodegen generate                # REQUIRED after adding/removing any source file
xcodebuild -project Shepherd.xcodeproj -scheme Shepherd -configuration Debug \
  -derivedDataPath ./build CODE_SIGNING_ALLOWED=NO \
  CLANG_MODULE_CACHE_PATH=./build/ModuleCache build
APP=./build/Build/Products/Debug/Shepherd.app
codesign --force --deep --sign - "$APP"   # ad-hoc sign so it runs locally
open "$APP"                                # relaunch: killall Shepherd; open "$APP"
```
First launch asks for notification permission ‚Äî allow it.

### 3. The Claude plugin (one-time)
```sh
ln -s "$PWD/claude-plugin" ~/.claude/skills/shepherd   # auto-loads as shepherd@skills-dir
```
Then `/reload-plugins` in any Claude session. Because it's symlinked, edits to
`claude-plugin/` apply on the next `/reload-plugins` ‚Äî no reinstall. It's a
**silent no-op outside Shepherd**, safe to leave installed globally.

## Architecture

**Rendering:** SwiftUI owns all chrome; **libghostty owns the terminal grid** ‚Äî
it renders via Metal into the `NSView` we hand it. A **tab is a container of a
`SplitNode` pane tree**; each **leaf pane** = one libghostty surface wrapped in
`NSViewRepresentable` = one tracked agent ([ADR 0012](.claude/adr/0012-pane-splitting-panes-as-agents.md)).
An unsplit tab is a single-leaf tree (one pane), behaving like the old
one-agent-per-tab model. `ContentView` renders the **selected** tab's tree via
the recursive `SplitContainer`. We never draw cells ourselves.

**The agent-state bridge (the "agent-native" part):**
```
claude (in a Shepherd pane, PTY has $SHEPHERD_TAB_ID + $SHEPHERD_SOCK injected)
  ‚Üí fires a lifecycle hook
  ‚Üí claude-plugin/hooks/report.sh  (pure bash; reads the env, sends JSON)
  ‚Üí unix socket ($SHEPHERD_SOCK)
  ‚Üí SocketServer (in-app)
  ‚Üí AgentStore.apply(event, detail, paneID)   # the lifecycle map, keyed per pane
  ‚Üí sidebar dot / dock badge / notification
```
Correlation is by the per-pane env var, not PID guessing. The env var NAME is
still `SHEPHERD_TAB_ID` (unchanged for plugin compatibility) but its **value is
the pane id** ‚Äî the plugin protocol and `report.sh` need no change. **Claude
Code is the only first-class agent** in v1 (see [ADR 0003](.claude/adr/0003-agent-state-via-claude-hooks.md)).

### Agent state lifecycle (`StopPolicy.applyEvent` + `AgentStore.apply`)
The transition map is the pure `applyEvent` ([ADR 0014](.claude/adr/0014-background-agent-stop-suppression.md));
`apply` is the shell that resolves the pane via `locatePane` (which walks
**every workspace**) and updates that pane's `AgentState` inside its tab's tree.
The dock badge, attention-nav (`‚åò‚áßA`), and notifications all **aggregate over
every pane across every tab in every workspace** (a tab's sidebar dot rolls its
panes up via `Tab.attentionState()`; `‚åò‚áßA`/`revealPane` switch workspace + tab +
focus to reach a hidden-workspace pane).

| Hook event | ‚Üí state |
|---|---|
| `SessionStart` | **idle** ; `SessionEnd` ‚Üí **shell** (agent gone, tab stays) |
| `UserPromptSubmit` | **working** (always ‚Äî starts a turn) |
| `PreToolUse[AskUserQuestion]` / `PreToolUse[ExitPlanMode]` | **blocked** ("answer needed" / "plan approval") |
| other `PreToolUse`/`PostToolUse`/`PostToolUseFailure`/`SubagentStart`/`SubagentStop`/`ElicitationResult` | **working** *(only if already mid-turn ‚Äî see guard)* |
| `PreToolUse[Agent]`/`[Task]` (mid-turn) | **working**, and `outstanding += 1` (background-agent counter) |
| `SubagentStop` | **working** if mid-turn; always `outstanding = max(0, outstanding ‚àí 1)` |
| `PermissionRequest` | **blocked** (+reason: "approve Bash" / "plan approval") |
| `Elicitation` | **blocked** ("input requested") |
| `Stop` | **need-to-check** ‚Äî *unless `outstanding > 0` (a background agent is still running), then held at **working** so a background-wait never reads as done ([ADR 0014](.claude/adr/0014-background-agent-stop-suppression.md))* |
| `StopFailure` | **error** (+reason: the API error type) |
| *focus / select tab / focus pane / click its notification* | need-to-check ‚Üí **idle** (only need-to-check; never blocked/working) ‚Äî `select()`/`focusPane()` call `didFocus` |

Plain panes (no agent) are **shell**. Sidebar currently shows **all** tabs
(filtering deferred ‚Äî [ADR 0006](.claude/adr/0006-sidebar-shows-all-tabs.md)).
Pane names: rename wins, else an agent's own OSC title, else the cwd ‚Äî the
priority now lives on the **pane** ([ADR 0011](.claude/adr/0011-tab-names-cwd-and-agent-title.md));
a tab's title is derived from its focused pane.

### Keybindings / menu commands (`ShepherdApp.swift`)
| Keys | Action |
|---|---|
| `‚åòT` | new tab (in the current workspace) |
| `‚åòW` | close the **focused pane** ‚Üí falls through to close-tab (last pane) ‚Üí **reseed a fresh tab** (last tab ‚Äî a workspace is never empty; no longer closes the window) |
| `‚åòD` | split right (`.row` ‚Äî side-by-side) |
| `‚åò‚áßD` | split down (`.column` ‚Äî stacked) |
| `‚åò‚áß‚Ü©` | toggle **zoom** of the focused pane (transient, not persisted) |
| `‚åò‚å•` + arrows | move pane focus directionally (geometric neighbor) |
| `‚åò‚áß[` / `‚åò‚áß]` | previous / next tab (within the current workspace) |
| `‚åò1`‚Äì`‚åò9` | jump to tab N (within the current workspace) |
| `‚åò‚áßA` | jump to next pane needing attention (blocked / need-to-check / error ‚Äî across **all** workspaces) |
| `‚åò‚áßN` | new workspace |
| `‚åÉ‚á•` / `‚åÉ‚áß‚á•` | next / previous workspace (wrap) |

A new pane inherits its parent pane's cwd. Splitting clears the tab's zoom.

### Sidebar (`SidebarView.swift`)
The sidebar header is the **workspace name** ‚Äî a custom dropdown (`WorkspaceSwitcher.swift`:
switch / inline-rename / delete-with-confirm / drag-reorder) plus a `+` to add a
workspace; a hover-gated two-finger horizontal swipe (`SidebarSwipe.swift`) cycles
workspaces (stopping at the ends) and the list **slides** directionally on switch.
Below the header, the tab list of the **current** workspace renders as before:
unsplit tabs render a single `TabRow`. A **split tab** renders a
`SplitTabGroup`: its panes gathered under a thin leading **bracket/rail** (a
rounded rail, not a curly `{`), one row per pane. The group **collapses** to a
`‚óè 1 ‚ñ∏ 2` strip of `<state-dot> <pane-number>` pips (hover a pip for its title);
collapse/expand toggles via a hover-only chevron. The focused/zoomed pane is
emphasized by **dimming the inactive panes** (no border ring) ‚Äî and a **zoomed**
pane dims its siblings in the sidebar too, in both the collapsed strip and the
expanded rows, so a tab reads as zoomed without switching to it. Collapse default
is a `shepherd.panes.defaultCollapsed` UserDefaults flag (ADR 0012 envisions
sourcing it from `~/.config/shepherd`; that wiring is a deferred follow-up).

### Persistence
The store key is now **`shepherd.workspaces.v1`** (`AgentStore`): per workspace,
its tabs (each a recursive `SplitNode` tree ‚Äî shape + split ratios ‚Äî plus each
pane's `userTitle` + cwd and the `collapsed` flag, in tab order) + the workspace
selection-by-index; the store also keeps `selectedWorkspaceID`. A one-time
migration wraps a legacy `shepherd.tabs.v2` blob into one default workspace.
Restore rebuilds the trees with **fresh pane ids and `.shell` state** (live agent
state and zoom never survive a restart; a restored pane is a fresh shell), and
re-derives selection from the persisted workspace/tab indices (ids regenerate).

## Critical gotchas (read before building / debugging)

- **macOS-26 toolchain** ([ADR 0002](.claude/adr/0002-libghostty-build-on-macos-26.md)): the ziglang.org Zig won't link (use brew `zig@0.15`); Metal shaders need the downloaded Metal Toolchain; build the xcframework only (`-Demit-macos-app=false`); Apple `libtool` dedupes same-named members so the combined fat archive is **incomplete** ‚Äî the build script fixes this with an `ld -r -all_load` merge of the constituent archives.
- **`report.sh` is pure bash on purpose** ([ADR 0004](.claude/adr/0004-plugin-protocol-and-ordering.md)). Do NOT add `python3` back ‚Äî its ~50ms startup delayed `PreToolUse` past `Stop` and flipped state. State is decided by event name + env; only 3 cosmetic events parse (via `jq`).
- **Ordering guard** ([ADR 0004](.claude/adr/0004-plugin-protocol-and-ordering.md)): mid-turn events only apply while the tab is `working`/`blocked`. A finished turn (`need-to-check`) is only left by a new `UserPromptSubmit` or focus. This is deliberate; don't remove it.
- **Background-agent `Stop` suppression** ([ADR 0014](.claude/adr/0014-background-agent-stop-suppression.md)): `Stop` fires *while a backgrounded agent is still running*, so a per-pane `outstanding` counter (`[Agent]`/`[Task]` launches ‚àí `SubagentStop`s, floored at 0) holds the pane at `working` instead of `need-to-check` when `outstanding > 0`. Don't gate this on `SubagentStart` ‚Äî it fires unreliably (seen 1 Start vs 6 Stops in one turn). Fails safe: a `Workflow` fan-out just reverts to finish-on-`Stop`.
- **Notification icon needs an asset catalog**: macOS Notification Center renders the app icon from the compiled asset catalog (`CFBundleIconName` ‚Üí `Assets.car`), **not** a loose `CFBundleIconFile`/`.icns` (which the Dock reads fine). `Resources/Assets.xcassets/AppIcon.appiconset` + `CFBundleIconName: AppIcon` in `project.yml` is what makes notifications show the Shepherd icon. After changing the icon, a stale system cache may need `lsregister` / a logout to refresh.
- **`AskUserQuestion` is detected via `PreToolUse`** ([ADR 0008](.claude/adr/0008-askuserquestion-via-pretooluse.md)) ‚Äî `PreToolUse[AskUserQuestion]` ‚Üí blocked. (`Elicitation`/`Notification` don't fire for it, but `PreToolUse` does ‚Äî `report.sh` parses `tool_name` via `jq`.)
- **`xcodegen generate` after any file add/remove** ‚Äî else the new file isn't compiled (`cannot find X in scope` at *build* time).
- **SourceKit lies in this repo** ‚Äî "Cannot find type AgentState/‚Ä¶" and "'main' attribute‚Ä¶" diagnostics are stale because the editor sees loose files, not the generated project. `xcodebuild` is ground truth; ignore SourceKit "cannot find" noise.
- **Debug log:** the app appends every state transition to `/tmp/shepherd-events.log` ‚Äî invaluable for debugging the state machine (`tail -f`). Currently always-on; a deferred cleanup is to put it behind a flag.
- **Sidebar is a custom `ScrollView`, NOT a `List`** ([ADR 0009](.claude/adr/0009-sidebar-custom-rows-not-list.md)): `List` was a keyboard-focus sink (stole keystrokes from the PTY) and forced ugly default styling. Don't reintroduce it ‚Äî and keep sidebar SwiftUI controls `.focusable(false)` so focus stays on the terminal.
- **`SplitAxis` vocab** ([ADR 0012](.claude/adr/0012-pane-splitting-panes-as-agents.md)): `.row` = `‚åòD` = panes **side-by-side**, **vertical** divider (`HStack`); `.column` = `‚åò‚áßD` = panes **stacked**, horizontal divider (`VStack`). "Row" means a row of panes, not a horizontal split ‚Äî easy to invert, so read it off here, not from intuition.
- **`store.tabs`/`selectedTab` are computed views of the *current* workspace** ([ADR 0013](.claude/adr/0013-workspaces.md)): the real owner is `workspaces` + `selectedWorkspaceID`. A pane in a hidden workspace is **not** in `store.tabs`, so socket/state feeds and attention/notification code must resolve panes via `locatePane(_:in: workspaces)` (or iterate `workspaces` directly) ‚Äî never assume a pane lives in the selected workspace.
- **Screenshotting the app for UI checks:** Shepherd often opens *behind* the frontmost window, so a full-screen `screencapture` grabs the wrong app. Capture its window directly ‚Äî get the id from `CGWindowListCopyWindowInfo` (a tiny `swift` script: owner == `Shepherd`, `kCGWindowLayer == 0`) ‚Üí `screencapture -o -l <id> out.png`. Re-`open` right after `killall` can hit LaunchServices `-600`; poll `until ! pgrep -x Shepherd` before reopening. (No `Quartz`/pyobjc on this machine ‚Äî use `swift`, not Python.)

## Conventions
- libghostty C API calls happen on the main thread.
- After editing `claude-plugin/`, the user runs `/reload-plugins` (symlinked).
- Don't commit `vendor/` (huge) or generated Xcode files ‚Äî they're gitignored.
- Match existing Swift style; keep comments to the non-obvious "why".
- UI colors live in `Theme.swift`; keep `GhosttyApp.writeBaseTheme()` in sync so the terminal grid matches the chrome ([ADR 0010](.claude/adr/0010-terminal-theme-from-shepherd-config.md)).
- Commit messages end with the project's Co-Authored-By line.

## Done vs deferred
**Done:** terminal (mouse/scroll/copy-paste/vsync/titles), tabs (create/switch/close/reorder/rename/persist+cwd, keyboard nav), **pane splitting** (H/V splits, zoom, draggable dividers, per-pane agents, bracket-grouped collapsible sidebar ‚Äî [ADR 0012](.claude/adr/0012-pane-splitting-panes-as-agents.md)), **workspaces** (Arc-style nested model, global cross-workspace attention + hidden-workspace notifications, name dropdown + `+`, two-finger swipe, ‚åò‚áßN/‚åÉ‚á•/‚åÉ‚áß‚á•, content cross-fade + sidebar slide, `shepherd.workspaces.v1` persistence w/ v2‚Üív1 migration ‚Äî [ADR 0013](.claude/adr/0013-workspaces.md)), agent-state lifecycle, Claude plugin, attention loop (badge + backgrounded notifications + ‚åò‚áßA jump-to-alert), **T3-Code-style sidebar** (custom rows not `List`, resizable, cwd/agent tab names) + self-contained theme (`~/.config/shepherd`), **sleep guard** ("Stay Awake" keep-awake with a 3-mode policy ‚Äî off / while-agents / always ‚Äî over `pmset disablesleep` Tier 2 with an IOKit idle-assertion Tier 1 fallback, 120s release grace, launch-reconcile + quit-teardown, clamshell display-blank + clamshell-gated thermal auto-sleep; pure `SleepPolicy` + optional passwordless-`pmset` sudoers setup in the README).
**Deferred (see SPEC ¬ß6):** generic non-Claude agents (Tier-B), sidebar auto-hide at ‚â§1 tab, debug-log flag, IME/selection polish, multi-window, navigator popup, and **full remote control** (the big future bet).
