# Shepherd ŌĆö agent-native macOS terminal

Shepherd is a native macOS terminal built on **libghostty** that treats **Claude
Code** sessions as first-class, tracked agents. It behaves exactly like a normal
terminal (iTerm/Ghostty-style) until you start `claude` in a tab ŌĆö then that tab
becomes an agent with a **live state** (working / blocked / need-to-check / idle /
error) surfaced in a sidebar, so you can run several at once and never babysit
them. When an agent needs you and you've switched apps, a notification + dock
badge pull you back.

**Status:** working v1 end-to-end (personal / OSS, single window). On `master`.
**Design doc:** [`SPEC.md`](SPEC.md). **Decisions:** [`.claude/adr/`](.claude/adr/) ŌĆö read these before changing anything load-bearing.

## How to build & run

### 1. libghostty (only needed once, or to update)
Prereqs (see [ADR 0002](.claude/adr/0002-libghostty-build-on-macos-26.md)): brew `zig@0.15`, Xcode + Metal Toolchain.
```sh
./scripts/build-libghostty.sh    # clones Ghostty v1.3.1, builds, vendors the xcframework
```

### 2. The app
```sh
cd spike/seam1
xcodegen generate                # REQUIRED after adding/removing any source file
xcodebuild -project Shepherd.xcodeproj -scheme Shepherd -configuration Debug \
  -derivedDataPath ./build CODE_SIGNING_ALLOWED=NO \
  CLANG_MODULE_CACHE_PATH=./build/ModuleCache build
APP=./build/Build/Products/Debug/Shepherd.app
codesign --force --deep --sign - "$APP"   # ad-hoc sign so it runs locally
open "$APP"                                # relaunch: killall Shepherd; open "$APP"
```
First launch asks for notification permission ŌĆö allow it.

### 3. The Claude plugin (one-time)
```sh
ln -s "$PWD/claude-plugin" ~/.claude/skills/shepherd   # auto-loads as shepherd@skills-dir
```
Then `/reload-plugins` in any Claude session. Because it's symlinked, edits to
`claude-plugin/` apply on the next `/reload-plugins` ŌĆö no reinstall. It's a
**silent no-op outside Shepherd**, safe to leave installed globally.

## Architecture

**Rendering:** SwiftUI owns all chrome; **libghostty owns the terminal grid** ŌĆö
it renders via Metal into the `NSView` we hand it. A **tab is a container of a
`SplitNode` pane tree**; each **leaf pane** = one libghostty surface wrapped in
`NSViewRepresentable` = one tracked agent ([ADR 0012](.claude/adr/0012-pane-splitting-panes-as-agents.md)).
An unsplit tab is a single-leaf tree (one pane), behaving like the old
one-agent-per-tab model. `ContentView` renders the **selected** tab's tree via
the recursive `SplitContainer`. We never draw cells ourselves.

**The agent-state bridge (the "agent-native" part):**
```
claude (in a Shepherd pane, PTY has $SHEPHERD_TAB_ID + $SHEPHERD_SOCK injected)
  ŌåÆ fires a lifecycle hook
  ŌåÆ claude-plugin/hooks/report.sh  (pure bash; reads the env, sends JSON)
  ŌåÆ unix socket ($SHEPHERD_SOCK)
  ŌåÆ SocketServer (in-app)
  ŌåÆ AgentStore.apply(event, detail, paneID)   # the lifecycle map, keyed per pane
  ŌåÆ sidebar dot / dock badge / notification
```
Correlation is by the per-pane env var, not PID guessing. The env var NAME is
still `SHEPHERD_TAB_ID` (unchanged for plugin compatibility) but its **value is
the pane id** ŌĆö the plugin protocol and `report.sh` need no change. **Claude
Code is the only first-class agent** in v1 (see [ADR 0003](.claude/adr/0003-agent-state-via-claude-hooks.md)).

### Agent state lifecycle (`StopPolicy.applyEvent` + `AgentStore.apply`)
The transition map is the pure `applyEvent` ([ADR 0015](.claude/adr/0015-background-stop-suppression-via-background-tasks.md));
`apply` is the shell that resolves the pane via `locatePane` (which walks
**every workspace**) and updates that pane's `AgentState` inside its tab's tree.
The dock badge, attention-nav (`ŌīśŌć¦A`), and notifications all **aggregate over
every pane across every tab in every workspace** (a tab's sidebar dot rolls its
panes up via `Tab.attentionState()`; `ŌīśŌć¦A`/`revealPane` switch workspace + tab +
focus to reach a hidden-workspace pane).

| Hook event | ŌåÆ state |
|---|---|
| `SessionStart` | **idle** (+ records the `session_id` from `detail` on the pane, for resume-on-relaunch) ; `SessionEnd` ŌåÆ **shell** (agent gone, tab stays; clears `sessionID`) |
| `UserPromptSubmit` | **working** (always ŌĆö starts a turn) |
| `PreToolUse[AskUserQuestion]` / `PreToolUse[ExitPlanMode]` | **blocked** ("answer needed" / "plan approval") |
| other `PreToolUse`/`PostToolUse`/`PostToolUseFailure`/`SubagentStart`/`SubagentStop`/`ElicitationResult` | **working** *(only if already mid-turn ŌĆö see guard)* |
| `PreToolUse[Agent]`/`[Task]` (mid-turn) | **working** ŌĆö no longer special-cased; background-ness is read from `Stop` ([ADR 0015](.claude/adr/0015-background-stop-suppression-via-background-tasks.md)) |
| `SubagentStop` | **working** if mid-turn (no longer adjusts a counter) |
| `PermissionRequest` | **blocked** (+reason: "approve Bash" / "plan approval") |
| `Elicitation` | **blocked** ("input requested") |
| `Stop` | **need-to-check** ŌĆö *unless the `Stop` payload's `background_tasks` still lists a backgrounded subagent/workflow/shell (a **monitor** does not count), then held at **working** so a background-wait never reads as done ([ADR 0015](.claude/adr/0015-background-stop-suppression-via-background-tasks.md))* |
| `StopFailure` | **error** (+reason: the API error type) |
| *focus / select tab / focus pane / click its notification* | need-to-check ŌåÆ **idle** (only need-to-check; never blocked/working) ŌĆö `select()`/`focusPane()` call `didFocus` |

Plain panes (no agent) are **shell**. Sidebar currently shows **all** tabs
(filtering deferred ŌĆö [ADR 0006](.claude/adr/0006-sidebar-shows-all-tabs.md)).
Pane names: rename wins, else an agent's own OSC title, else the cwd ŌĆö the
priority now lives on the **pane** ([ADR 0011](.claude/adr/0011-tab-names-cwd-and-agent-title.md));
a tab's title is derived from its focused pane.

A new pane inherits its parent pane's cwd. Splitting clears the tab's zoom.

### Sidebar (`SidebarView.swift`)
The sidebar header is the **workspace name** ŌĆö a custom dropdown (`WorkspaceSwitcher.swift`:
switch / inline-rename / delete-with-confirm / drag-reorder) plus a `+` to add a
workspace; a hover-gated two-finger horizontal swipe (`SidebarSwipe.swift`) cycles
workspaces (stopping at the ends) and the list **slides** directionally on switch.
Below the header, the tab list of the **current** workspace renders as before:
unsplit tabs render a single `TabRow`. A **split tab** renders a
`SplitTabGroup`: its panes gathered under a thin leading **bracket/rail** (a
rounded rail, not a curly `{`), one row per pane. The group **collapses** to a
`ŌŚÅ 1 Ō¢Ė 2` strip of `<state-dot> <pane-number>` pips (hover a pip for its title);
collapse/expand toggles via a hover-only chevron. The focused/zoomed pane is
emphasized by **dimming the inactive panes** (no border ring) ŌĆö and a **zoomed**
pane dims its siblings in the sidebar too, in both the collapsed strip and the
expanded rows, so a tab reads as zoomed without switching to it. Collapse default
is a `shepherd.panes.defaultCollapsed` UserDefaults flag (ADR 0012 envisions
sourcing it from `~/.config/shepherd`; that wiring is a deferred follow-up).

### Persistence
The store key is now **`shepherd.workspaces.v1`** (`AgentStore`): per workspace,
its tabs (each a recursive `SplitNode` tree ŌĆö shape + split ratios ŌĆö plus each
pane's `userTitle` + cwd + `sessionID` and the `collapsed` flag, in tab order) +
the workspace selection-by-index; the store also keeps `selectedWorkspaceID`. A
one-time migration wraps a legacy `shepherd.tabs.v2` blob into one default workspace.
Restore rebuilds the trees with **fresh pane ids and `.shell` state** (live agent
state and zoom never survive a restart), and re-derives selection from the persisted
workspace/tab indices (ids regenerate).

**Agent resume:** a pane persists the live Claude `sessionID` (captured from the
`SessionStart` hook payload via `detail`; cleared on `SessionEnd`). On restore, if a
pane carries one, `GhosttyTerminal.makeSurface` seeds libghostty's `initial_input` with
`claude --resume <id>\n` (`AgentStore.takeResumeInput` ŌåÆ `claudeResumeInput`) so the
agent picks up in the already-restored cwd ŌĆö composes with the `shepherdd pty` wrapper
since it's typed into the PTY, not the surface command. One-shot (id consumed async on
read); a live resume re-arms it via the resumed session's own `SessionStart`, a dead id
falls back to a plain shell. Requires the plugin reporting `session_id` ŌĆö after editing
`report.sh`, run `/reload-plugins`; sessions started before the reload won't resume.

## Critical gotchas (read before building / debugging)

- **macOS-26 toolchain** ([ADR 0002](.claude/adr/0002-libghostty-build-on-macos-26.md)): the ziglang.org Zig won't link (use brew `zig@0.15`); Metal shaders need the downloaded Metal Toolchain; build the xcframework only (`-Demit-macos-app=false`); Apple `libtool` dedupes same-named members so the combined fat archive is **incomplete** ŌĆö the build script fixes this with an `ld -r -all_load` merge of the constituent archives.
- **`report.sh` is pure bash on purpose** ([ADR 0004](.claude/adr/0004-plugin-protocol-and-ordering.md)). Do NOT add `python3` back ŌĆö its ~50ms startup delayed `PreToolUse` past `Stop` and flipped state. State is decided by event name + env; only 3 cosmetic events parse (via `jq`).
- **Ordering guard** ([ADR 0004](.claude/adr/0004-plugin-protocol-and-ordering.md)): mid-turn events only apply while the tab is `working`/`blocked`. A finished turn (`need-to-check`) is only left by a new `UserPromptSubmit` or focus. This is deliberate; don't remove it.
- **Background-`Stop` suppression** ([ADR 0015](.claude/adr/0015-background-stop-suppression-via-background-tasks.md)): `Stop` fires *while a backgrounded agent is still running*, so `report.sh` reduces the `Stop` payload's `background_tasks` (Claude Code v2.1.145+) to a count of tasks the turn is paused on ŌĆö allow-list **subagent/workflow/shell** hold the notification; a **monitor** does not ŌĆö and `applyEvent` holds the pane at `working` when that count > 0. This **replaces** ADR 0014's launch-vs-`SubagentStop` counter, which was unreliable because those events don't pair 1:1 (seen 1 `Start` vs 6 `Stop`s). Fails safe: empty/unparseable `detail` ŌåÆ treated as 0 ŌåÆ plain finish-on-`Stop`. Decide payload questions from the **raw payload**, not `/tmp/shepherd-events.log` (it logs transitions, not payloads ŌĆö the false premise behind ADR 0014).
- **Notification icon needs an asset catalog**: macOS Notification Center renders the app icon from the compiled asset catalog (`CFBundleIconName` ŌåÆ `Assets.car`), **not** a loose `CFBundleIconFile`/`.icns` (which the Dock reads fine). `Resources/Assets.xcassets/AppIcon.appiconset` + `CFBundleIconName: AppIcon` in `project.yml` is what makes notifications show the Shepherd icon. After changing the icon, a stale system cache may need `lsregister` / a logout to refresh.
- **`AskUserQuestion` is detected via `PreToolUse`** ([ADR 0008](.claude/adr/0008-askuserquestion-via-pretooluse.md)) ŌĆö `PreToolUse[AskUserQuestion]` ŌåÆ blocked. (`Elicitation`/`Notification` don't fire for it, but `PreToolUse` does ŌĆö `report.sh` parses `tool_name` via `jq`.)
- **`xcodegen generate` after any file add/remove** ŌĆö else the new file isn't compiled (`cannot find X in scope` at *build* time).
- **SourceKit lies in this repo** ŌĆö "Cannot find type AgentState/ŌĆ”" and "'main' attributeŌĆ”" diagnostics are stale because the editor sees loose files, not the generated project. `xcodebuild` is ground truth; ignore SourceKit "cannot find" noise.
- **Debug log:** the app appends every state transition to `/tmp/shepherd-events.log` ŌĆö invaluable for debugging the state machine (`tail -f`). Currently always-on; a deferred cleanup is to put it behind a flag.
- **Sidebar is a custom `ScrollView`, NOT a `List`** ([ADR 0009](.claude/adr/0009-sidebar-custom-rows-not-list.md)): `List` was a keyboard-focus sink (stole keystrokes from the PTY) and forced ugly default styling. Don't reintroduce it ŌĆö and keep sidebar SwiftUI controls `.focusable(false)` so focus stays on the terminal.
- **`SplitAxis` vocab** ([ADR 0012](.claude/adr/0012-pane-splitting-panes-as-agents.md)): `.row` = `ŌīśD` = panes **side-by-side**, **vertical** divider (`HStack`); `.column` = `ŌīśŌć¦D` = panes **stacked**, horizontal divider (`VStack`). "Row" means a row of panes, not a horizontal split ŌĆö easy to invert, so read it off here, not from intuition.
- **`store.tabs`/`selectedTab` are computed views of the *current* workspace** ([ADR 0013](.claude/adr/0013-workspaces.md)): the real owner is `workspaces` + `selectedWorkspaceID`. A pane in a hidden workspace is **not** in `store.tabs`, so socket/state feeds and attention/notification code must resolve panes via `locatePane(_:in: workspaces)` (or iterate `workspaces` directly) ŌĆö never assume a pane lives in the selected workspace.
- **Screenshotting the app for UI checks:** Shepherd often opens *behind* the frontmost window, so a full-screen `screencapture` grabs the wrong app. Capture its window directly ŌĆö get the id from `CGWindowListCopyWindowInfo` (a tiny `swift` script: owner == `Shepherd`, `kCGWindowLayer == 0`) ŌåÆ `screencapture -o -l <id> out.png`. Re-`open` right after `killall` can hit LaunchServices `-600`; poll `until ! pgrep -x Shepherd` before reopening. (No `Quartz`/pyobjc on this machine ŌĆö use `swift`, not Python.)

## Conventions
- libghostty C API calls happen on the main thread.
- After editing `claude-plugin/`, the user runs `/reload-plugins` (symlinked).
- Don't commit `vendor/` (huge) or generated Xcode files ŌĆö they're gitignored.
- Match existing Swift style; keep comments to the non-obvious "why".
- UI colors live in `Theme.swift`; keep `GhosttyApp.writeBaseTheme()` in sync so the terminal grid matches the chrome ([ADR 0010](.claude/adr/0010-terminal-theme-from-shepherd-config.md)).
- Commit messages end with the project's Co-Authored-By line.

## Done vs deferred
**Done:** terminal (mouse/scroll/copy-paste/vsync/titles), tabs (create/switch/close/reorder/rename/persist+cwd, keyboard nav), **pane splitting** (H/V splits, zoom, draggable dividers, per-pane agents, bracket-grouped collapsible sidebar ŌĆö [ADR 0012](.claude/adr/0012-pane-splitting-panes-as-agents.md)), **workspaces** (Arc-style nested model, global cross-workspace attention + hidden-workspace notifications, name dropdown + `+`, two-finger swipe, ŌīśŌć¦N/ŌīāŌćź/ŌīāŌć¦Ōćź, content cross-fade + sidebar slide, `shepherd.workspaces.v1` persistence w/ v2ŌåÆv1 migration ŌĆö [ADR 0013](.claude/adr/0013-workspaces.md)), agent-state lifecycle, Claude plugin, attention loop (badge + backgrounded notifications + ŌīśŌć¦A jump-to-alert), **T3-Code-style sidebar** (custom rows not `List`, resizable, cwd/agent tab names) + self-contained theme (`~/.config/shepherd`), **sleep guard** ("Stay Awake" keep-awake with a 3-mode policy ŌĆö off / while-agents / always ŌĆö over `pmset disablesleep` Tier 2 with an IOKit idle-assertion Tier 1 fallback, 120s release grace, launch-reconcile + quit-teardown, clamshell display-blank + clamshell-gated thermal auto-sleep; pure `SleepPolicy` + optional passwordless-`pmset` sudoers setup in the README).
**FCM push (Android Phase 1 step 2):** attention transitions route to local surfaces
(banner+sound) when present, or a data-only FCM wake when away (`isAway` = lid shut +
